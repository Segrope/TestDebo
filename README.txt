GITHUB

Repository : 
Dossier qui contient le projet, comme par exemple les fichiers (cachés par défaut sur Mac) qui sont utilisés pour communiquer avec Git.

Ajouter un repo : drag & drop le dossier dans l’application. Si c’est un dossier git, tout l’historique sera importé et connecté avec les comptes Github. Sinon, Github proposera de créer un nouveau repo depuis celui-ci.

______

Travailler avec les repositories

Chaque personne peut avoir une copie d’un repository

Forking : 
Pour synchroniser des changements locaux sur un repository distant, il faut les permissions de lecture et d’écriture « push & pull » 

Pour obtenir le push & pull permission, il faut « fork » le projet Github à son propre compte.


Exemple : 
On fork un projet en cliquant sur le bouton « Fork » : signifie qu’on va créer une branche au nom de l’utilisateur sur ce projet. On a donc fait « bifurquer » le projet Github sur mon compte.


Depuis le compte Github, on peut désormais cloner le projet pour obtenir directement la branche sur l’application Github. On a cloné le projet.

 ___________


Réaliser des changements

On chope le repository dans la sidebar. Open In Finder
On ouvre un fichier contenu dans le repository. On réalise une modification, puis sauvegarde.

On a effectué une modification sur la copie locale. On va utiliser Github pour commit ce changement.

Un commit est une photographie du projet (codes, fichiers, etc…) à un point particulier dans le temps.
__________

En retournant sur l’app Github, la modification apparaît. Cela signifie qu’une modification a eu lieu sur la copie locale. Pour commiter, on fait un bref résumé du changement et on clique sur « Commit to master » 

= Nous avons donc commité sur le Git localement mais pas encore synchronisé avec le repository de github.com

Pour synchroniser sur github, suffit de cliquer sur le bouton Synchronisation en haut à droite
